# FFT大小选择功能说明

## 功能概述

用户现在可以自由选择FFT大小，从512到32768，应用会自动处理累积缓冲区和重叠窗口，确保大FFT也能正常工作。

## 使用方法

### FFT大小选择

1. **打开FFT大小选择对话框**
   - 点击底部控制栏的"FFT大小"按钮
   - 会弹出选择对话框

2. **选择FFT大小**
   - 对话框显示所有可用的FFT大小选项
   - 每个选项显示：
     - FFT大小值
     - 频率分辨率（Hz）
     - 性能提示（优秀/良好/延迟较高/高延迟）

3. **确认选择**
   - 选择后立即生效
   - 如果选择≥16384，会显示性能警告
   - 设置会自动保存，下次启动时恢复

### 重叠率选择（提高刷新率）

1. **打开重叠率选择对话框**
   - **长按**底部控制栏的"FFT大小"按钮
   - 会弹出重叠率选择对话框

2. **选择重叠率**
   - **无重叠 (0%)**: 最低CPU占用，但更新频率最低
   - **低重叠 (25%)**: 平衡性能和刷新率
   - **中重叠 (50%)**: **默认推荐**，最佳平衡
   - **高重叠 (75%)**: 最高刷新率，但CPU占用最高

3. **重叠率的影响**
   - 重叠率越高，更新频率越高（刷新更快）
   - 但CPU占用也更高（需要更频繁计算FFT）
   - 对于大FFT（如32768），提高重叠率可以显著改善用户体验

## 技术实现

### 1. 累积缓冲区
- **小FFT (≤音频buffer大小)**: 直接处理，无需累积
- **大FFT (>音频buffer大小)**: 使用累积缓冲区，收集足够的样本后再处理

### 2. 重叠窗口
- 使用50%重叠窗口
- 提高更新频率，减少延迟感
- 确保频谱连续性

### 3. 自动保存
- 使用SharedPreferences保存用户选择
- 应用重启后自动恢复上次设置

### 4. 性能警告
- FFT大小≥16384时显示橙色警告
- FFT大小≥8192时显示浅橙色提示
- 选择大FFT时会弹出确认对话框

## 使用建议

### 实时音频分析
- **推荐**: 2048-4096
- **最佳**: 2048（默认）
- **不推荐**: >8192（延迟明显）

### 高精度分析
- **推荐**: 8192-16384
- **可接受**: 32768（建议使用75%重叠率提升刷新率）

### 快速预览
- **推荐**: 512-1024
- **优势**: 低延迟，流畅显示

## 注意事项

1. **大FFT延迟**: FFT大小越大，更新频率越低，延迟越明显
   - **解决方案**: 使用高重叠率（75%）可以显著提升刷新率
2. **内存占用**: 大FFT需要更多内存（32768约3.2MB）
3. **CPU占用**: 
   - 大FFT计算量大，可能影响设备性能
   - **重叠率越高，CPU占用越高**（需要更频繁计算FFT）
   - 75%重叠率下，CPU占用约为无重叠的4倍
4. **实时性**: 
   - 32768配合75%重叠率可以达到约5.4 FPS，基本满足实时分析需求
   - 如果设备性能不足，可以降低重叠率
5. **重叠率选择**: 
   - 小FFT（≤4096）：50%重叠率足够
   - 大FFT（≥8192）：建议使用75%重叠率
   - 如果设备发热或卡顿，可以降低重叠率

## 重叠率功能说明

### 什么是重叠率？

重叠率（Overlap Ratio）是指相邻两个FFT窗口之间的重叠比例。

- **0%重叠**: 每个FFT窗口完全独立，无重叠
- **50%重叠**: 每个新窗口包含前一个窗口50%的数据
- **75%重叠**: 每个新窗口包含前一个窗口75%的数据

### 重叠率的优势

1. **提高更新频率**: 重叠率越高，更新频率越高
2. **改善频谱连续性**: 重叠窗口使频谱变化更平滑
3. **减少时间分辨率损失**: 对于瞬态信号检测更有利

### 重叠率的代价

1. **CPU占用增加**: 需要更频繁计算FFT
2. **内存占用略增**: 需要维护更大的缓冲区

### 实际效果示例（32768 FFT）

| 重叠率 | 更新频率 | CPU占用 | 用户体验 |
|--------|---------|---------|---------|
| 0%     | ~1.3 FPS | 低      | 卡顿明显 |
| 25%    | ~1.8 FPS | 中低    | 略有改善 |
| 50%    | ~2.7 FPS | 中      | 可接受   |
| 75%    | ~5.4 FPS | 高      | **流畅** |

**结论**: 对于32768这样的大FFT，75%重叠率可以将更新频率提升4倍，显著改善用户体验。
